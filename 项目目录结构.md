```
poetry-platform/
├── client-miniprogram/      # 微信小程序端
```

## 九、技术栈总结

### 推荐技术栈组合

#### 方案一：全栈JavaScript/TypeScript（推荐）

**前端**：
- 小程序：uni-app（可同时编译小程序和H5）
- 管理端：Vue 3 + Element Plus + Vite

**后端**：
- 框架：NestJS（TypeScript）
- 数据库：MySQL 8.0 + Redis 6+
- 搜索：Elasticsearch 7+
- ORM：TypeORM
- 实时通信：Socket.io

**优势**：
- 全栈使用JavaScript/TypeScript，降低学习成本
- NestJS结构清晰，易于维护
- uni-app可以一套代码多端运行

#### 方案二：前端Vue + 后端Java

**前端**：同方案一

**后端**：
- 框架：Spring Boot
- 数据库：MySQL + Redis
- ORM：MyBatis Plus
- 搜索：Elasticsearch

**优势**：
- Java生态成熟，性能稳定
- 适合大型项目和团队协作

#### 方案三：前端React + 后端Python

**前端**：
- 小程序：Taro（React语法）
- H5：React + Next.js
- 管理端：React + Ant Design

**后端**：
- 框架：FastAPI（高性能）
- 数据库：MySQL + Redis
- ORM：SQLAlchemy

**优势**：
- Python简洁易读
- FastAPI性能优秀，自动生成API文档

## 十一、目录结构设计原则

1. **模块化**：按功能模块组织代码，便于维护
2. **分层清晰**：MVC/MVVM架构，职责分离
3. **可扩展**：易于添加新功能，不影响现有代码
4. **规范统一**：命名规范、目录结构规范一致
5. **工程化**：使用构建工具、代码规范检查、自动化测试
